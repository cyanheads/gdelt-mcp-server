# 0.1.8 — 2026-06-04

## Fixed

- **`GdeltTvService.buildBaseParams`**: multiple `stations` entries now join with `OR` inside parentheses — `(station:CNN OR station:MSNBC)` — matching the GDELT TV API requirement. Space-separated selectors caused a deterministic API error for any call with two or more stations. Affects `gdelt_search_tv`, `gdelt_get_tv_clips`, and `gdelt_get_tv_context`. ([#8](https://github.com/cyanheads/gdelt-mcp-server/issues/8))
- **`GdeltTvService.getTvContext`**: `clipsAnalyzed` no longer defaults to `0` when `numclips` is absent from the upstream wordcloud response. The field is now optional and omitted when the API provides no value — prevents misleading "0 clips analyzed" alongside hundreds of valid co-occurrence terms. ([#13](https://github.com/cyanheads/gdelt-mcp-server/issues/13))
- **`gdelt_get_tv_context` enrichment**: `totalCount` is now optional in the output schema; `ctx.enrich.total()` is guarded and only called when `clipsAnalyzed` is present. ([#13](https://github.com/cyanheads/gdelt-mcp-server/issues/13))
- **`gdelt_get_tv_trending` `no_trending` recovery**: replaced misleading "Retry after a short delay" with a redirect to `gdelt_search_tv` with explicit date bounds — the TV archive is permanently frozen at October 2024, so retrying is never useful. ([#11](https://github.com/cyanheads/gdelt-mcp-server/issues/11))
