# 0.2.0 — 2026-06-08

## Added

- **`gdelt_get_tv_context` `startDatetime`/`endDatetime` inputs** — explicit date-range targeting, matching every other TV tool. `getTvContext` now calls `applyTimeRange()` instead of setting `timespan` manually. ([#15](https://github.com/cyanheads/gdelt-mcp-server/issues/15))
- **`resolveTimespan` and `formatDateShort` helpers** exported from `gdelt-fetch.ts` — parse GDELT timespan strings (`1y`, `6m`, `7d`, `24h`, `15min`) to absolute date ranges for display in error recovery hints. ([#12](https://github.com/cyanheads/gdelt-mcp-server/issues/12))
- **Resolved-range notice in TV tool error paths** — when `timespan` resolves outside the archive window and returns empty, `gdelt_search_tv`, `gdelt_get_tv_clips`, and `gdelt_get_tv_context` now include the computed date span in the recovery hint (e.g., `Timespan "1y" resolved to 2025-06-08 – 2026-06-08`). ([#12](https://github.com/cyanheads/gdelt-mcp-server/issues/12))
- **Cap-hit `notice` on `gdelt_search_articles` and `gdelt_get_tv_clips`** — when `result.length >= maxRecords`, `ctx.enrich.notice()` fires with the count and a prompt to increase `maxRecords`. ([#15](https://github.com/cyanheads/gdelt-mcp-server/issues/15))
- **`startDatetime`/`endDatetime` echo on DOC API tools** — `gdelt_get_coverage_timeline`, `gdelt_get_tone_distribution`, and `gdelt_get_coverage_breakdown` now echo these fields in `enrichment` when supplied, enabling agents to verify their date window was honored. ([#15](https://github.com/cyanheads/gdelt-mcp-server/issues/15))

## Fixed

- **`GdeltRateLimiter.acquire()` abort-signal propagation** — accepts an optional `AbortSignal`; rejects immediately when already aborted and splices the queued entry on abort, unblocking subsequent callers rather than holding the slot for the full delay. `gdeltFetch` passes `ctx.signal` through. ([#9](https://github.com/cyanheads/gdelt-mcp-server/issues/9))

## Changed

- **`devcheck.config.json` `outdated.allowlist`** — `@cyanheads/mcp-ts-core` added to hold at `0.9.21`.
- **`@types/node`** `^25.9.1 → ^25.9.2`
